# レーススカウト 毎日更新手順（自動更新エージェント用）

毎朝7:00(JST)に実行し、①前日結果の集計 ②当日の地方競馬・競輪・ボートレース・オートレース全レースのAI予想生成 を行って公開する。

## 手順

1. 今日の日付を `YYYYMMDD` 形式で確認する（JST基準）。
2. まず前回分の結果を集計する（未集計の過去日は自動で処理される）:
   ```
   python3 results.py --auto
   ```
   - 「結果未確定」で保留された日は翌朝また自動で試行されるので放置してよい。
   - 同じ日が2日以上「結果未確定」のまま残る場合はレース中止の可能性が高いので
     `python3 results.py <日付> --force` で確定分だけ集計する。
3. データ収集を実行する:
   ```
   python3 scrape.py YYYYMMDD
   ```
   - 種別ごとに `data/YYYYMMDD_<type>.json` ができる。開催のない種別はスキップされる。
   - 全種別「開催なし」の場合は、結果集計分だけcommit/pushして終了してよい。
   - **当日分のみ**を予想・公開する（翌日分を先に作らない）。
4. `data/YYYYMMDD_<type>.json` を読み、**全レースを予想**して `predictions/tmp_<type>_<会場ローマ字>.json` を会場ごとに書く。フォーマットは下記「予想JSONフォーマット」を厳守。
   - レース数が多い（1日250〜300R）。**サブエージェントを「種別×会場」単位で並列に使うこと**（1会場=1ファイル）。
   - 各サブエージェントには、担当会場のdataのJSON部分・出力ファイルパス・下記フォーマットと分析観点を渡す。
5. マージしてサイト用JSONを生成する:
   ```
   python3 build.py YYYYMMDD
   ```
   - 「警告: 予想が見つからないレース」が出たら、そのレースの予想を追加してからやり直す。
6. `predictions/tmp_*.json` を削除し、コミットしてpushする:
   ```
   rm predictions/tmp_*.json
   git add -A
   git commit -m "predict: YYYYMMDD"
   git push
   ```
7. push後、https://rossyrich.github.io/race-scout/ に反映される（GitHub Pagesのビルドに1〜2分）。

## 予想JSONフォーマット（predictions/tmp_<type>_<会場>.json）

```json
{
 "type": "boat",
 "venue": "住之江",
 "races": [
  {
   "race_id": "B20260720_12_01",
   "no": 1,
   "marks": [
    {"mark": "◎", "num": 1, "name": "選手名", "comment": "本命理由を30字程度"},
    {"mark": "◯", "num": 3, "name": "選手名", "comment": "..."},
    {"mark": "▲", "num": 4, "name": "選手名", "comment": "..."},
    {"mark": "△", "num": 2, "name": "選手名", "comment": "..."}
   ],
   "summary": "展開予想と狙いを2〜3文",
   "confidence": "A",
   "bets": {
    "nirenpuku": ["1-3", "1-4"],
    "nirentan": ["1-3", "1-4", "3-1"],
    "sanrenpuku": ["1-3-4", "1-2-3", "1-3-5"],
    "sanrentan": ["1→3→4", "1→3→2", "1→4→3", "3→1→4"]
   }
  }
 ]
}
```

- `race_id`・`no`・`num`・`name` は data のJSONの値をそのまま使う。
- `confidence`: S(鉄板) / A(有力) / B(混戦) / C(荒れ模様)
- 全レース分を必ず出力（12R開催なら12レース）。

### marks の数

| 種別 | 印 |
|---|---|
| 地方競馬 | ◎◯▲△△ の5頭（8頭未満なら4頭） |
| 競輪(7車) | ◎◯▲△△ の5車（5車立てなら4車） |
| ボート(6艇) | ◎◯▲△ の4艇 |
| オート(8車) | ◎◯▲△△ の5車 |

### bets のキーと点数

| 種別 | キー | 点数 |
|---|---|---|
| keiba | tansho / umaren / sanrenpuku / sanrentan | 単勝1点・**馬連3〜5点(主軸)**・三連複4〜6点・三連単4〜8点 |
| keirin | nirenpuku / nirentan / sanrenpuku / sanrentan | ２車複2〜3点・２車単2〜4点・３連複3〜5点・**３連単4〜8点(主軸)** |
| boat | 同上 | ２連複2〜3点・２連単2〜4点・３連複3〜5点・**３連単4〜8点(主軸)** |
| auto | 同上 | 同上 |

- 表記: 順不同系は "1-3"、着順指定系は "1→3→4"。番号は馬番/車番/艇番。

### 種別ごとの分析観点

- **地方競馬**: 過去5走の着順・クラス・タイム、騎手、斤量、脚質、間隔、中央からの転入実績、距離適性、オッズ(取れた場合)。帯広ばんえいは負担重量と障害(第2障害)の実績を重視。
- **競輪**: **ライン構成の想定が最重要**（同支部・同県・同期でラインを組む。dataのline情報があれば使い、なければ府県・期別・脚質から自分で想定し、summaryに書く）。級班・競走得点・脚質(逃/両/追)・決まり手・直近成績。ラインの先頭(自力型)の強さと番手の差し脚を評価。
- **ボート**: **1コースの1着率が最重要**(全国平均55%前後)。級別(A1>A2>B1>B2)・全国/当地勝率・モーター2連率・平均ST・今節成績。イン逃げ濃厚ならS、センター勢の捲り警戒なら評価を下げる。
- **オート**: **ハンデ差の構造が最重要**。最重ハンデ(最後方スタート)の実力者が捲り切れるかを軸に、級別・平均競走タイム・当地実績・直近着順で評価。

## 運用ルール

- 予想は**当日朝のみ**公開（前日先出ししない）。未公開日のタブは「朝8時ごろまでに公開」表示になる。
- 成績タブ = 種別ごとに「自信度×券種の勝敗 + 回収率 + 日別結果」を累計表示（results/DATE.json から自動生成）。
- ミッドナイト競輪(〜23:25頃)があるため、結果集計は翌朝に行う設計。
